## 认证与权限

### JWT Token

- **格式**：Bearer Token（Authorization header）
- **有效期**：24 小时
- **签名方式**：HS256
- **密钥**：来自 `JWT_SECRET` 环境变量

### 权限检查

所有 API 接口（除了 `/api/auth/login`）都需要：
1. 有效的 JWT token（通过 JWTMiddleware）
2. 对应的菜单权限（通过 PermissionMiddleware）

权限检查基于用户的 Role 和 Role 关联的 Menu。

### DataScope（数据范围过滤）

系统支持四层数据权限范围：

| DataScope | 说明 | 角色示例 |
|-----------|------|----------|
| school | 学校级全权限 | 校长、教务 |
| college | 学院级权限 | 院长 |
| major | 专业/阶段级权限 | 专业主任 |
| class | 班级级权限 | 讲师 |

DataScope 在登录时写入 JWT claims，查询用户/部门等数据时自动按 DataScope 过滤。

## 常见问题

### Q: 如何修改 JWT 密钥？
A: 修改 `.env` 文件中的 `JWT_SECRET`。生产环境建议使用强密钥。

### Q: 如何添加新的权限检查？
A: 在 `internal/router/router.go` 的对应路由上添加 `middleware.PermissionMiddleware("menu-id")`。

### Q: 如何扩展数据模型？
A: 在 `internal/model/` 中添加新的 GORM 结构体，GORM 会自动建表（如果开启 AutoMigrate）。
